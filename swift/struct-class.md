## Struct vs Class

이번 글에선, Struct와 Class에 대해 알아보고 둘을 비교 해보겠다. 물론 내가 이해한 방향대로 적는거라 정확하지 않을 수 있다.

우선 공통점을 알아보겠습니다.

- 값을 저장할 프로퍼티를 선언할 수 있습니다. (값을 저장할 변수를 선언할 수 있습니다)
- 메소드를 선언할 수 있다. (내부 함수를 만들 수 있습니다)
- 내부 프로퍼티에 .을 사용하여 접근 할 수 있습니다.
- 생성자를 사용해 초기 상태를 설정할 수 있습니다.
- Protocol, extension의 사용이 가능합니다.

차이점은 아래와 같습니다.

- class는 레퍼런스 타입으로 인해 heap 에 있으며, 클래스는 여러군데서 참고해도 결국 하나만이 힙 영역에 남아있고, struct는 stack에 있고 고유의 값이 각각 참조하지 못하는 형태로 들어가게 됩니다.
- class는 상속이 가능합니다.
- 같은 클래스 인스턴스 만들고 값 변경시 모든 변수에 영향을 끼칩니다.
- ARC의 참조 횟수 계산은 클래스에만 작동합니다. (레퍼런스 타입이므로)

---

코드

```swift
struct SomeStructure {
    // 내용
}
class SomeClass {
    // 내용
}

//우선 위와 같이 정의 해줄 수 있습니다


```
